# Project Brief: H.A.C.A - Home Assistant Config Auditor

## Project Overview
H.A.C.A (Home Assistant Config Auditor) is a custom Home Assistant integration designed to analyze and improve Home Assistant configurations. It provides automated auditing, health scoring, and refactoring capabilities for Home Assistant installations.

## Core Purpose
To help Home Assistant users maintain healthy, secure, and high-performing configurations by:
- Analyzing configuration files for best practices violations
- Identifying security risks and performance issues
- Providing automated refactoring tools
- Generating comprehensive health reports

## Key Components
- **Specialized Analyzers**: Automation, Performance, Security, and Entity Health analyzers
- **AI Integration**: Conversational explanations and personalized advice
- **Refactoring Assistant**: Automated fixes with backup capabilities
- **Web Interface**: Custom panel for issue management and reporting
- **Diagnostic Sensors**: Real-time health monitoring

## Technical Stack
- **Language**: Python 3.10+
- **Framework**: Home Assistant Custom Component
- **Dependencies**: PyYAML, fpdf2
- **Frontend**: Custom JavaScript panel
- **Storage**: Home Assistant configuration storage

## Project Status
- **Version**: 1.1.0
- **Status**: Active development
- **Distribution**: HACS (Home Assistant Community Store)
- **License**: MIT

## Target Users
- Home Assistant users with complex configurations
- Users concerned about security and performance
- Those wanting automated configuration maintenance
- Advanced users managing multiple automations and entities

## Success Metrics
- Configuration health score improvement
- Reduction in configuration errors
